# 条件触发规则

> **说明**：条件触发规则是"必须执行"，不是可选。当简历信息命中条件时，对应维度的题目必须包含施压/追问类问题，覆盖常规问题之外。

| 条件                                 | 触发动作                   | 示例                                                         |
| ------------------------------------ | -------------------------- | ------------------------------------------------------------ |
| 学历为硕士以下（本科/大专/高中）     | 学历题必须追问学历天花板   | "你目前是本科，现在硕士遍地走，你觉得自己的学历上限在哪里？怎么弥补？" |
| 学历为博士                           | 学历题转向研究方向深度     | "博士期间你的研究方向挺专的，但企业更看重落地能力，你怎么平衡？" |
| 学校非985/211/双一流                 | 学校题必须追问学校背书问题 | "你的学校在行业内知名度不算高，你觉得这会影响你在求职中的竞争力吗？" |
| 学校为985/211/双一流                 | 学校题转向个人能力证明     | "名校光环确实加分，但面试中我更想听到你个人做出了什么不一样的事？" |
| 期望薪资高于行业平均水平             | 薪资题施压                 | "你要的这个数在同级别里偏高，你觉得自己哪一点值这个价？"     |
| 期望薪资低于行业平均水平             | 薪资题追问底气             | "你要的比市场价低一截，是底气不足还是对岗位理解有偏差？"     |
| 个人优点仅抽象描述（如"学习能力强"） | 追问具体证据               | "你说了学习能力强，能不能给一个最近半年学会新东西的真实例子？" |
| 个人缺点过于圆滑（如"太追求完美"）   | 追问真实短板               | "这个回答我们耳朵都听出茧了，能不能说一个真的缺点——那种会影响工作的？" |
| 项目经历描述偏宏观缺少个人贡献       | 追问个人角色               | "你刚才把项目讲得挺大，但我没听出来你具体干了什么？你在里面到底承担什么角色？" |
| 技能列表中出现"精通"等绝对词         | 追问深度                   | "你写了精通XXX，如果满分10分你给自己打几分？为什么不是满分？" |

## 执行方式

Agent 在 Phase1 生成问题阶段必须读取本文件。Phase2 评估阶段不需要读取。
